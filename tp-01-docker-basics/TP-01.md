# TP-01: Docker Basics

**Difficulty**: Beginner 🟢  
**Estimated Duration**: 30 min  

## Learning Objectives
- Verify Docker installation
- Understand the difference between images and containers
- Pull images from Docker Hub
- Run, stop, and list containers
- Inspect logs and execute commands inside containers

## Prerequisites
- Docker Engine installed and running on your host (Linux/WSL2).
- Basic terminal knowledge.

---

## Instructions

### Step 1: Verify Installation
Let's start by making sure Docker is correctly installed and the daemon is responding.

```bash
docker version
docker info
```
*Explanation*: 
- `docker version` displays the client and server version. If you only see the client, the Docker daemon might not be running or your user doesn't have privileges.
- `docker info` provides system-wide information like the number of containers and images, and the storage driver.

### Step 2: Download your first image
An **image** is a read-only template used to create containers. We will pull (download) the lightweight `alpine` Linux image.

```bash
docker pull alpine:latest
docker images
```
*Explanation*: `docker pull` fetches the image from the registry (Docker Hub by default). `docker images` lists all images stored locally on your machine.

### Step 3: Run your first container
A **container** is a running instance of an image. Let's start an `alpine` container and execute the `sh` (shell) command interactively.

```bash
docker run -it alpine sh
```

Once inside the container, try a few commands:
```bash
/ # cat /etc/os-release
/ # echo "Hello from Docker" > greeting.txt
/ # exit
```
*Explanation*: 
- `-i` (interactive) and `-t` (tty) allocate a pseudo-TTY and keep STDIN open, allowing you to interact with the shell.
- When you type `exit`, the process running as PID 1 inside the container ends, and the container automatically stops.

### Step 4: Manage containers in the background
Sometimes we want containers to keep running in the background. Let's start an `nginx` web server.

```bash
docker run -d --name my-web-server -p 8080:80 nginx:alpine
```
*Explanation*:
- `-d` runs the container in detached mode (in the background).
- `--name` assigns a custom name (`my-web-server`).
- `-p 8080:80` maps port 8080 on your host machine to port 80 inside the container.

Test the server by opening your browser at `http://localhost:8080` or using `curl`:
```bash
curl http://localhost:8080
```

### Step 5: List, inspect, and execute
Check which containers are running:
```bash
docker ps
```
To see stopped containers as well, use `docker ps -a`.

View the logs generated by your web server:
```bash
docker logs my-web-server
```

Execute a command inside the already running container (e.g., open a shell):
```bash
docker exec -it my-web-server sh
```
Inside, run `ls /usr/share/nginx/html` then `exit`.

### Step 6: Cleanup
Let's stop and remove the container.
```bash
docker stop my-web-server
docker rm my-web-server
docker ps -a
```
*Explanation*: You must stop a running container before you can remove it (unless you force it with `docker rm -f`).

If you want to remove the downloaded image:
```bash
docker rmi nginx:alpine
```
