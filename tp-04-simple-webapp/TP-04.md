# TP-04: Simple Web App Stack

**Difficulty**: Intermediate 🟡  
**Estimated Duration**: 45 min  

## Learning Objectives
- Orchestrate a multi-container architecture manually
- Forward environment variables inside run commands
- Run a backend REST API connecting to a Redis database
- Realize the complexity of managing stacks without Compose

## Prerequisites
- Completed TP-01, TP-02, and TP-03
- Python basic knowledge (optional)

---

## Instructions

This lab simulates a realistic microservices pattern. We have a Python backend connecting to a Redis Database to count visitors. 
We will do this manually to appreciate what `docker-compose` solves for us later.

### Step 1: Examine the Application
Look at `app/app.py`. It uses the `redis` python package and connects to a `REDIS_HOST` environment variable.

### Step 2: Create a network
These two applications need to talk to each other. Remember TP-03? We need a user-defined network!

```bash
docker network create my-app-net
```

### Step 3: Run the Database Container
Run the official Redis image. We'll name the container `redis-server`, run it in background (`-d`), and attach it to our network. No need to map ports locally (no `-p`) because the Python app will reach it over the internal network.

```bash
docker run -d --name redis-server --network my-app-net redis:alpine
```

### Step 4: Build the Python Application
Navigate to `app/` and build the Python application image.
(Create the `Dockerfile` inside the `tp-04-simple-webapp/` folder if it doesn't exist already! See contents below)

Let's assume the `Dockerfile` in `tp-04-simple-webapp` is structured similarly to TP-02:
```bash
docker build -t counter-app:v1 .
```

### Step 5: Run the App Container
This is the critical step. We need to pass the `REDIS_HOST` environment variable to match the database container name (`redis-server`).

```bash
docker run -d --name web-app -p 8080:5000 \
    --network my-app-net \
    -e REDIS_HOST=redis-server \
    counter-app:v1
```
*Explanation*: The `-e REDIS_HOST=redis-server` injects an environment variable into the container. Because both containers are on `my-app-net`, the DNS will resolve `redis-server` to the correct IP.

### Step 6: Test the Application
Open a browser or use curl to access `http://localhost:8080`.
Refresh it several times.
```bash
curl http://localhost:8080
```
*Expected Output*: `{"message":"Hello! Visitors count: 1"}`, incrementing every request.

**Notice how annoying this is?** 
If the application had 5 services (Frontend, API, Worker, Database, Cache), you would need a long script to start everything in the right order with the correct networks and environment variables. Welcome to the "Dependency Hell" that Docker Compose solves.

### Step 7: Cleanup
```bash
docker stop web-app redis-server
docker rm web-app redis-server
docker network rm my-app-net
```
