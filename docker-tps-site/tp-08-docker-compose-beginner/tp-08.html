TP-08 : Débuter avec Docker Compose

Débutant ⏱ Durée estimée : 30 min

🎯 Objectifs d'apprentissage

  • Remplacer l'enchaînement fastidieux des docker run par un unique fichier déclaratif YAML.
  • Définir une "Stack" multi-conteneurs basique (serveur web + file de cache redis).
  • Utiliser un fichier .env externe pour déporter la configuration dynamique.

📝 Étape par étape

Nous allons recréer le scénario du TP-04, mais cette fois-ci géré avec l'orchestrateur local Docker Compose.

Étape 1 : Lecture Pédagogique du Fichier

Ouvrez et contemplez l'élégance de docker-compose.yml et de son acolyte caché .env.

  • Services : Deux briques, `web` (votre API) et `redis`. Compose sait que l'un a besoin de l'autre grâce à depends_on.
  • Réseau Absolu : Avez-vous vu une commande de création de réseau ? Non ! Compose génère automatiquement un réseau de pile (Stack Network) privé à l'instant où on le lance, et y connecte tous vos services implicitement.
  • Variables secrètes/config : Le port exposé de l'API est tiré automatiquement de la variable WEB_PORT lue dans le fichier .env ! Oubliez la gestion manuelle.

Étape 2 : Un Seul Anneau Pour Les Gouverner Tous

Lancez purement et simplement toute l'architecture en arrière-plan depuis le dossier du TP :

docker compose up -d

Vérifiez le sous-groupe :

docker compose ps

Étape 3 : Suivi d'Opérations Unifiées

Plus besoin de taper chaque ID de conteneur, demandez les journaux unifiés en streaming :

docker compose logs -f

Et seulement pour la base de données :

docker compose logs redis

Étape 4 : Destruction Systémique

Le développeur arrête de travailler : suppression propre en un clic :

docker compose down

Tous les conteneurs sont stoppés, supprimés, et l'immense réseau par défaut est nettoyé silencieusement.
