TP-12 : La Stack d'Expert "Production-Ready" Intégrale

Expert ⏱ Durée estimée : 60 min

🎯 Objectifs d'apprentissage

  • Orchestrer l'ensemble des concepts vu dans un point d'orgue de qualité totale.
  • Lire et assimiler la puissance prodigieuse du Multi-Stage Build pour occulter les dépendances compilées avant livraison de l'image minimale (Frontend Alpine -> Nginx Image).
  • Manipuler un conteneur Proxy inverse de type Engine x (Nginx) sans jamais percer les sécurités DB / Backend vers internet public.

📝 Étape par étape

L'apothéose DevOps, ou une tentative de rapprochment extrême du standard du marché :
Frontend public -> Firewall(Proxy IN) -> Backend(Non Root) -> Database(Cache+SQL) .

Étape 1 : Audit Global visuel de 10 Minutes (Tâches du Senior)

Fouillez votre répertoire courant et validez chaque composante ! L'Outil est robuste :

  • frontend/Dockerfile : Contemplez le "AS builder". L'mage pèse très lourd, elle va compiler (imaginons Node ou Webpack), et dans une 2nd temps (`FROM Nginx`) un autre disque propre copiera uniquement le build résultant via --from=builder. C'est l'essence du cloud moderne sans surpoid de faille de sécurité.
  • backend/Dockerfile : Le linter n'a rien à dire. Compilation par étapes. Création d'un groupe applicatif pour l'exécution d'interface.
  • config/nginx.conf : C'est ce fichier vital mis en route par le Conteneur Proxy qui distribue vers `/` pour la page html Frontend, mais route sur l'URL de base réseau-privée Docker nommé `/api` la requête du navigateur client pour contacter Postgres Indirectement.

Étape 2 : Simulation CI Automatisée Intransigeante

Chez les pro, pas de "docker compose UP aveugle". Le CI Pipeline de GitLab passera tout ça à la moulinette scanner Hadolint Linter + Scanner de Sécurité Trivy du socle O.S image Alpine, etc...

Testons le script Bash préparé par notre lead Dev : 

chmod +x scripts/ci-pipeline.sh
./scripts/ci-pipeline.sh

Résultat : Il vous lintera sans pitié les deux images, bâtira localement la fondation intermédiaire, puis lancera TRIVY Aquasec pour garantir la version zéro vulnérabilité critique. Tout est VERRE DE GAUCHE : Déploiement Autorisé.

Étape 3 : "Go Live !" Déploiement de Stack

docker compose up -d

Dégustez au visuel les "Depends_on / Conditions_healthy" fonctionner par rebond et effet domino asynchrone ! La base initialise son disque persistant, validé. Python Backend API valide sa boucle 10s sur db... Puis ENFIN Le Proxy Web frontal nginx, rassuré d'une arborescence complète derrière lui, donne le feu-vert extérieur.

Étape 4 : Test Réel de charge Proxy Extérieure

Seul et unique trou de douane dans l'infrastructure : le port 80 de proxy.

  • Navigateur Local (Front) : `http://localhost/` (Route NGINX vers votre index de web)
  • Interface JSON (Back) : `http://localhost/api/count` (Route NGINX vers le Flask qui incrémente sans exposer la bdd !)

Étape 5 : L'Ultime Clap de Fin 🎉

docker compose down -v

🚀 Félicitations d'Achevement Extrême

Vous êtes passés des tristes `Docker Run` enchaînés à tâtons en ligne bash, aux puissants fichiers déclaratifs `Docker Compose` blindés de healthchecks, de réseaux clos sans IPv4 explicites, avec une propreté logicielle prouvable via Linter / Scanner intégration continue et des compilations Multi-Phase à micro-poids d'exploitation !
