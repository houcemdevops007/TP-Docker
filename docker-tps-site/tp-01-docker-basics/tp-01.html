TP-01 : Les Bases de Docker

Débutant ⏱ Durée estimée : 30 min

🎯 Objectifs d'apprentissage

  • Vérifier l'installation de Docker sur votre machine.
  • Comprendre la différence fondamentale entre une image et un conteneur.
  • Télécharger des images depuis le Docker Hub.
  • Lancer, arrêter et lister des conteneurs.
  • Inspecter les journaux (logs) et exécuter des commandes à l'intérieur de conteneurs actifs.

🛠 Prérequis

  • Docker Engine installé et fonctionnel (Linux ou WSL2 sous Windows).
  • Connaissances très basiques d'un terminal Linux.

📝 Étape par étape

Étape 1 : Vérifier l'installation

Commençons par nous assurer que le client Docker est bien installé et que le service (daemon) répond correctement.

docker version
docker info

Explication : docker version affiche la version du client et du serveur. docker info donne des détails système globaux comme l'espace de stockage et le nombre de conteneurs existants.

Étape 2 : Télécharger votre première image

Une image est un modèle en lecture seule utilisé pour créer des conteneurs. Nous allons "tirer" (télécharger) la minuscule image Linux alpine.

docker pull alpine:latest
docker images

Explication : docker pull va chercher l'image sur le registre par défaut (Docker Hub). docker images liste toutes les images stockées localement sur votre disque.

Étape 3 : Lancer votre premier conteneur

Un conteneur est une instance en cours d'exécution d'une image. Lançons un conteneur alpine tout en ouvrant un shell (invite de commande) à l'intérieur.

docker run -it alpine sh

Une fois à l'intérieur du conteneur, tapez ces commandes :

cat /etc/os-release
echo "Bonjour depuis Docker" > message.txt
exit

Explication : Les options -i (interactif) et -t (pseudo-TTY) vous lient au terminal du conteneur. Lorsque vous tapez exit, le processus principal s'arrête, ce qui arrête automatiquement le conteneur complet.

Étape 4 : Les conteneurs en arrière-plan (Détachés)

Souvent, nous voulons qu'un serveur web tourne en tâche de fond. Essayons avec le serveur web nginx.

docker run -d --name mon-serveur-web -p 8080:80 nginx:alpine

Explication :

  • -d (detached) : exécute le conteneur en arrière-plan.
  • --name : donne un nom personnalisé lisible (sinon Docker en invente un).
  • -p 8080:80 : très important ! Cela redirige le port 8080 de votre machine physique vers le port 80 (par défaut d'nginx) à l'intérieur du conteneur.

Testez le serveur web via le navigateur à http://localhost:8080 ou via le terminal :

curl http://localhost:8080

Étape 5 : Gestion au quotidien

Lister les conteneurs actuellement en cours d'exécution :

docker ps

Pour voir tous les conteneurs (même ceux qui sont arrêtés), utilisez docker ps -a.

Voir les logs générés par le serveur web lorsqu'il a reçu votre requête :

docker logs mon-serveur-web

Exécuter une nouvelle commande (ex: ouvrir un shell) dans un conteneur qui tourne *déjà* (contrairement à run) :

docker exec -it mon-serveur-web sh

Tapez ls /usr/share/nginx/html puis exit.

Étape 6 : Nettoyage

Il est temps d'arrêter et de supprimer le conteneur.

docker stop mon-serveur-web
docker rm mon-serveur-web
docker ps -a
