# TP-08: Docker Compose Beginner

**Difficulty**: Beginner 🟢  
**Estimated Duration**: 30 min  

## Learning Objectives
- Replace complex `docker run` commands with a single YAML file
- Define a multi-container stack (web + database)
- Use an `.env` file to manage environment variables

## Prerequisites
- Completed TP-04 (so you know why Compose is useful!)
- Docker Compose v2 installed (`docker compose version`)

---

## Instructions

We will recreate the scenario from TP-04, but using Docker Compose instead.

### Step 1: Examine the Configuration
Open `docker-compose.yml` and `.env`.

Notice how the `docker-compose.yml` file elegantly defines:
- **Services**: `web` and `redis`
- **Networks**: Not explicitly defined! Compose creates a default network automatically and attaches both services to it.
- **Volumes**: Not explicitly defined yet, but available.
- **Environment variables**: Pulled automatically from the `.env` file!

### Step 2: Start the Stack
In the terminal, inside the `tp-08-docker-compose-beginner` directory simply run:

```bash
docker compose up -d
```
*Explanation*: 
- `up` creates and starts the networks, volumes, and containers.
- `-d` runs them in the background (detached mode).

Check the running services:
```bash
docker compose ps
```

### Step 3: Test the Application
Even though the `web` container is using the `counter-app:v1` image from TP-04, if you didn't build it, Compose would fail. If you deleted it, quickly run `docker build -t counter-app:v1 ../tp-04-simple-webapp/`.

Open your browser to `http://localhost:8080`.

### Step 4: View the Logs
You don't need to specify individual container names anymore to see logs for the whole app.

```bash
docker compose logs -f
```
Press `Ctrl+C` to stop tailing the logs.

To see logs for just one service:
```bash
docker compose logs redis
```

### Step 5: Execute Commands
To run a command inside a specific service, use `docker compose exec`.
Let's see what's inside the cache:

```bash
docker compose exec redis redis-cli get hits
```

### Step 6: Teardown
To cleanly stop all containers and remove the default network created by Compose:

```bash
docker compose down
```

Try changing the port binding in `docker-compose.yml` from `8080:5000` to `8081:5000` and restart the stack using `docker compose up -d`. Compose will automatically recreate *only* the web container!
