# TP-07: Image Security Scan with Trivy

**Difficulty**: Advanced 🔴  
**Estimated Duration**: 30 min  

## Learning Objectives
- Understand container vulnerabilities and supply chain security
- Run Trivy to scan an intentionally vulnerable Docker image
- Use Trivy filters to fail a CI pipeline on HIGH or CRITICAL severity
- Remediate vulnerabilities by updating base images

## Prerequisites
- Completed TP-02
- `trivy` installed locally OR available via Docker

---

## Instructions

We'll scan an older, intentionally vulnerable Dockerfile (`Dockerfile.vulnerable`) and then fix it (`Dockerfile.fixed`).

### Step 1: Examine the vulnerable Dockerfile
Open `Dockerfile.vulnerable`. 
Notice it uses the `ubuntu:18.04` base image, which is extremely old and contains thousands of unpatched vulnerabilities.

Let's build it:
```bash
docker build -f Dockerfile.vulnerable -t secure-app:vulnerable .
```

### Step 2: Scan the image with Trivy
If installed locally:
```bash
trivy image secure-app:vulnerable
```

Using Docker:
```bash
docker run --rm -v /var/run/docker.sock:/var/run/docker.sock \
    aquasec/trivy image secure-app:vulnerable
```

*Expected output*: You should see a giant table containing hundreds of vulnerabilities (CVEs) with severities from UNKNOWN to CRITICAL.

### Step 3: Filter Critical Vulnerabilities
In a CI pipeline, you usually don't want to break the build for every minor issue. You only want to fail on HIGH or CRITICAL vulnerabilities that have a known fix.

Run the provided script:
```bash
chmod +x scripts/run-trivy.sh
./scripts/run-trivy.sh secure-app:vulnerable
```
Notice the script uses: `--severity HIGH,CRITICAL --exit-code 1 --ignore-unfixed`.
Since the image is vulnerable, the script will exit with code 1.

### Step 4: Fix the Image
Open `Dockerfile.fixed`. 
This file uses a highly optimized and secure base image (`alpine:3.18`), which has a tiny attack surface.

Build and scan the new image:
```bash
docker build -f Dockerfile.fixed -t secure-app:fixed .
./scripts/run-trivy.sh secure-app:fixed
```

*Expected output*: No vulnerabilities found! The exit code is 0.

### Step 5: Clean up
```bash
docker rmi secure-app:vulnerable secure-app:fixed
```

## Check your understanding
1. Why is scanning the base image so important for security?
2. What does `--ignore-unfixed` do in Trivy, and why is it useful in a CI pipeline?
3. Why did `alpine` have significantly fewer (or zero) vulnerabilities compared to `ubuntu:18.04`?
