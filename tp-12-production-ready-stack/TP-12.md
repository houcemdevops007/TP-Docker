# TP-12: Production-Ready Stack (Expert)

**Difficulty**: Expert 🟢🔴  
**Estimated Duration**: 60 min  

## Learning Objectives
- Review and combine all previous concepts into one realistic stack.
- Build multi-stage Dockerfiles.
- Configure a reverse proxy (Nginx).
- Understand how a CI pipeline (Linting + Security + Build) fits together.

## Prerequisites
- Completed ALL previous TPs.

---

## Instructions

Welcome to the final lab. This is a complete, production-grade microservice architecture. 

It contains:
1. A **Frontend** (Nginx serving static HTML via multi-stage build).
2. A **Backend** API (Python Flask), running as a non-root user.
3. A **Database** (PostgreSQL) with persistent volume.
4. A **Reverse Proxy** (Nginx) routing traffic securely using custom `nginx.conf`.
5. **Networks**: `front-net` and `back-net` for isolation.
6. **Healthchecks** and Resource Limits for resiliency.
7. A complete mock **CI/CD script** tying linting, scanning, and building together.

### Step 1: Examine the Codebase
Take 10 minutes to review the following files:
- `frontend/Dockerfile` (Notice the multi-stage build: `builder` -> `runner`).
- `backend/Dockerfile` (Notice the `hadolint` compliant structure and `nonroot` user).
- `config/nginx.conf` (Notice how it proxies `/api` to the backend and `/` to the frontend).
- `docker-compose.yml` (Look at the network definitions, depends_on conditions, and env vars).

### Step 2: Run the CI Pipeline Script
Before deploying, a true DevOps engineer validates the codebase. We have provided `scripts/ci-pipeline.sh` that mocks a CI pipeline.

Make it executable and run it:
```bash
chmod +x scripts/ci-pipeline.sh
./scripts/ci-pipeline.sh
```
The script will sequentially:
1. Lint the Dockerfiles with `hadolint`
2. Build the images
3. Scan the images with `trivy`

If any step fails, the pipeline halts. Ensure it passes!

### Step 3: Start the Stack
Bring everything up:
```bash
docker compose up -d
docker compose ps
```
Wait a few seconds for the database and backend healthchecks to pass.

### Step 4: Test the Architecture
The only port exposed to the host machine is port 80 (via the Proxy).

- Browse to `http://localhost`: You should see the Frontend UI.
- Browse to `http://localhost/api/count`: You should see the Backend API response, incrementing via the Database!

### Step 5: Teardown and Congratulations
```bash
docker compose down -v
```

🎉 **Congratulations! You have completed the Docker & Docker Compose Training Track!** 🎉
