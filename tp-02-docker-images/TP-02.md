# TP-02: Building First Images with Dockerfile

**Difficulty**: Beginner 🟢  
**Estimated Duration**: 45 min  

## Learning Objectives
- Write a `Dockerfile` using common instructions: `FROM`, `RUN`, `COPY`, `CMD`, `EXPOSE`
- Build a custom Docker image from the `Dockerfile`
- Tag and run the custom image

## Prerequisites
- Completed TP-01
- Python installed (optional, but we will use a Python app in the container)

---

## Instructions

For this lab, we provide a simple Python Flask application in the `app/` folder.

### Step 1: Examine the application
Take a look at `app/app.py` and `app/requirements.txt`.
The app is a simple web API that returns "Hello DevOps" on port 5000.

### Step 2: Write the Dockerfile
In the `tp-02-docker-images/` folder, create a file named `Dockerfile` right next to the `app/` directory.

Add the following content to the `Dockerfile`:

```dockerfile
# Step 1: Define the base image. Using a lightweight python image.
FROM python:3.9-slim

# Step 2: Set the working directory inside the container
WORKDIR /usr/src/app

# Step 3: Copy only the requirements file first (caching optimization)
COPY app/requirements.txt ./

# Step 4: Install Python dependencies
RUN pip install --no-cache-dir -r requirements.txt

# Step 5: Copy the rest of the application code
COPY app/ ./

# Step 6: Expose the port the app runs on
EXPOSE 5000

# Step 7: Define the command that will run when the container starts
CMD ["python", "app.py"]
```
*Explanation*: 
- We copy `requirements.txt` and install dependencies before copying the application code. Docker caches layers, so if we only change the code and not the dependencies, the build is much faster!

### Step 3: Build the Docker Image
Now, we instruct Docker to build an image based on the instructions inside the `Dockerfile`.

```bash
docker build -t hello-devops:v1 .
```
*Explanation*:
- `docker build` builds the image.
- `-t hello-devops:v1` tags the image with a name (`hello-devops`) and a label/version (`v1`).
- `.` specifies the context directory that contains the `Dockerfile` and the files to copy.

Verify that the image was created:
```bash
docker images | grep hello-devops
```

### Step 4: Run your custom image
Start a container using your newly built image.

```bash
docker run -d --name my-python-app -p 8080:5000 hello-devops:v1
```

Test that the app is responding natively on your host on port `8080`:
```bash
curl http://localhost:8080
```
*Expected Output*: `{"message":"Hello DevOps ver. 1.0"}` (or similar depending on your app code).

Check the logs to see the Flask server output:
```bash
docker logs my-python-app
```

### Step 5: Modifying and Rebuilding (Image Tags)
Edit the `app/app.py` file to return "Hello DevOps (V2)" instead.
Then, rebuild the image with a new tag:

```bash
docker build -t hello-devops:v2 .
```
Notice how fast the `RUN pip install` step went? That's the Docker build cache in action!

Run a new container on another port to compare:
```bash
docker run -d --name my-python-app-v2 -p 8081:5000 hello-devops:v2
curl http://localhost:8081
```

### Step 6: Pushing to a Registry (Optional)
To share your image, you would tag it with your docker registry username and push it:
```bash
# docker login
# docker tag hello-devops:v2 yourusername/hello-devops:v2
# docker push yourusername/hello-devops:v2
```

### Step 7: Cleanup
```bash
docker stop my-python-app my-python-app-v2
docker rm my-python-app my-python-app-v2
```
