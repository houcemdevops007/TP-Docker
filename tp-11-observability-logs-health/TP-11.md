# TP-11: Observability, Logs, and Health

**Difficulty**: Advanced 🔴  
**Estimated Duration**: 30 min  

## Learning Objectives
- Use logging drivers in Docker Compose (specifically `json-file` with limits)
- Implement an application-layer healthcheck endpoint
- Configure Docker Compose to poll an HTTP `/health` endpoint

## Prerequisites
- Completed TP-10

---

## Instructions

We have a Python application in the `app/` folder that exposes a `/health` endpoint. This endpoint returns a `503 Service Unavailable` status code for the first 10 seconds, and then a `200 OK`.

### Step 1: Examine the Application and Compose File
Open `app/app.py` to see the logic.
Then, open `docker-compose.yml`.

Notice two major additions for the `api` service:
1. **Logging configuration**:
   ```yaml
   logging:
     driver: "json-file"
     options:
       max-size: "10m"
       max-file: "3"
   ```
   This prevents the container from generating gigabytes of logs and filling up your disk (a very common production issue!). It rotates the logs, keeping only 3 files of 10MB each.

2. **HTTP Healthcheck**:
   ```yaml
   healthcheck:
     test: ["CMD", "wget", "--spider", "-q", "http://localhost:5000/health"]
     interval: 3s
     timeout: 3s
     retries: 5
   ```
   Instead of `CMD-SHELL`, we directly use `wget --spider` to request the endpoint. If it returns 200, it succeeds. If it returns 503, it fails.

### Step 2: Start and Monitor
Open your terminal and start the stack in detached mode:
```bash
docker compose up -d --build
```

Immediately run `docker compose ps` repeatedly:
```bash
watch -n 1 "docker compose ps"
```
*(Use `Ctrl+C` to exit watch).*

You will see `api` start in `(health: starting)`. It will stay that way (or temporarily `(unhealthy)`) until 10 seconds pass, at which point it becomes `(healthy)`.

### Step 3: View Logs
Now send some traffic to the application:
```bash
curl http://localhost:8080/
curl http://localhost:8080/health
```

Check the logs.
```bash
docker compose logs -f api
```
Notice how Docker Compose automatically tags the log lines with the service name (`api-1`). 

### Step 4: Teardown
```bash
docker compose down
```
